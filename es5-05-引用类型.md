1. Array: shift() //删除第一项

		  unshift(xxx,xxx); //在数组前面加数据

		  sort(); //默认的是从小到大排列，不过是把所有元素转为String之后再比较，
		  		  //所以会出现 2 10 5 这种输出，所以一般给sort加一个比较函数
		  	例如：function compare(v1, v2){
		  			if(v1 < v2){
		  				return -1;
		  			} else if(v1 > v2){
		  				return 1;
		  			} else {
		  				return 0;
		  			}
		  		 }
		  		 array.sort(compare); //从小到大输出

		  reverse(): 反转一个数组

		  slice(start, end):end可选。返回从start到end的元素，start为负数的话是从后开始(-1 -2 ...)。此函数不会改变原数组

		  splice() //向数组中删除/添加/替换元素。此函数可以此数组
		  splice(start, num); //删除从start开始算起，num个元素
		  splice(start, 0, xxx,yyy,zzz...);// 插入xxx yyy zzz等项
		  splice(start, num, xx, yy, zz...); //删除num个元素后，插入xx yy zz...

		  indexOf(); //从头查找，返回要查找的参数的第一次出现的位置，没有的话返回-1
		  lastIndexOf(); //从尾查找，返回要查找的参数第一次出现的位置，没有的话返回-1

		  every() ：对数组中的每一项运行给定函数，如果该函数对每一项都返回 true ，则返回 true 。 
		  			常用于判断数组中的元素是否满足某条件
		  some() ：对数组中的每一项运行给定函数，如果该函数对任一项返回 true ，则返回 true 。 
		  			常用于判断数组中的元素是否满足某条件
		  filter() ：对数组中的每一项运行给定函数，返回该函数会返回 true 的项组成的数组。 
		  			用于返回符合某条件的元素
		  map() ：对数组中的每一项运行给定函数，返回每次函数调用的结果组成的数组。

		  reduce()和reduceRight(): 前者是从前往后，后者是从后往前。用法是函数返回的任意值都会传给下一项，依次迭代。
		  例子：array.reduce(function(pre, cur, index, array){
		  			return pre + cur
				})
		  以上方法都不会修改数组中的包含的值。

3. Function类型：
		js里函数没有重载，相同名字的函数，后面的会覆盖前面的

		函数内部有arguments和this两个属性，arguments是一个类数组，里面包含的是传入的参数，一般不要在代码里用这个变量，显示的传变量
		this指的是可以执行的环境对象，不通的环境下指向的对象是不一样的(常用的是：本类或window)

		Function有两个属性：length和prototype。
		length：希望接受到的参数数量。
		prototype：保存了函数所有的实例方法，比如下面的apply和call

		apply和call：都是在特定的作用域中调用函数，不同的只是传递的参数不一样
		apply(运行函数的作用域, 参数数组);
		call(运行函数的作用域, param1, param2, param3等等);
		例子：
			window.color = 'red';
			var o = {color:blue};
			function sayColor(){
				alert(this.color);
			}
			sayColor() // red
			sayColor.call(this); // red
			sayColor.call(window); // red
			sayColor.call(o); // blue

		bind：会创建一个新函数，成为绑定函数。 
			let newFunc = func.bind(作用域); 
			其意义是:绑定函数会以创建它时传入bind()方法的第一个参数作为this作用域.
			可以这么理解：func函数里 有/要使用 this变量，这样的话this的指向就是传入bind的作用域了
			用在creator里常常是用在回调函数里
